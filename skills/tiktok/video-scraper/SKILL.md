# tiktok-video-scraper

Scrape TikTok public content: user profiles, hashtag videos, search results, and trending feed — **no TikTok account required** for most features. User's own video list requires a valid session cookie from a logged-in Chrome browser.

---

## ⚠️ Critical Context

TikTok's API has **asymmetric protection**:

| What you want | Works without login? | How |
|--------------|---------------------|-----|
| User profile stats | ✅ Yes | SSR HTML extraction |
| Hashtag videos (paginated) | ✅ Yes | `challenge/item_list` API |
| Search results (paginated) | ✅ Yes | `search/general/full` API |
| Trending feed | ✅ Yes | `recommend/item_list` API |
| User's own video list | ❌ No | Requires sessionid cookie |
| Comments | ❌ No | Requires sessionid cookie |
| Private profiles | ❌ No | Requires sessionid + follow |

**Why user videos require auth**: TikTok signs `post/item_list` requests with an `X-Bogus` token computed by obfuscated JavaScript. Without this signature, the API returns HTTP 200 but an empty response body. The signature can only be generated by the browser's own JS — you cannot forge it.

**Bot detection**: TikTok uses PerimeterX + in-house bot scoring. Requests from a real Chrome browser via CDP are NOT detected as bots. Direct curl/HTTP requests will be blocked.

**No WAF for public endpoints**: `challenge/item_list`, `search/general/full`, and `recommend/item_list` all work without X-Bogus. The bot detection on these is relatively lenient.

- Node.js 22+

## Setup (For User Video Scraping)

### Step 1: Capture session from Chrome

Make sure Chrome is open and you're **logged in to TikTok** at `https://www.tiktok.com`.

If Chrome has remote debugging enabled (port 9333 or 9222):
```bash
node video-scraper.mjs auth --cdp-url=http://localhost:9333
# or:
node video-scraper.mjs auth --cdp-url=http://localhost:9222
```

Session is saved to: `~/.local/share/showrun/data/tiktok/session.json`

### Step 2: Verify session
```bash
node video-scraper.mjs check-session
```

### Step 3: Scrape user videos
```bash
node video-scraper.mjs user-videos charlidamelio --count=30 --pages=5
```

## Usage

### Scrape a public profile (no login)
```bash
node video-scraper.mjs profile <username>
node video-scraper.mjs profile @charlidamelio   # @ prefix is stripped automatically
```

### Scrape hashtag videos (no login, paginated)
```bash
node video-scraper.mjs hashtag <hashtag>
node video-scraper.mjs hashtag #funny          # # prefix is stripped automatically
node video-scraper.mjs hashtag dance --count=30 --pages=5
```

### Search videos (no login, paginated)
```bash
node video-scraper.mjs search <keyword>
node video-scraper.mjs search "dance tutorial" --count=20 --pages=3
```

### Get trending feed (no login)
```bash
node video-scraper.mjs trending --count=20
```

### Scrape user's own videos (requires auth)
```bash
node video-scraper.mjs user-videos <username> --count=30 --pages=5
```

### Save results to file
```bash
node video-scraper.mjs hashtag cats --output=/tmp/cats.json
node video-scraper.mjs profile khaby.lame --output=/tmp/khaby.json
```

### Use specific Chrome instance
```bash
node video-scraper.mjs profile charlidamelio --cdp-url=http://localhost:9333
```

### Show browser window (useful for debugging)
```bash
node video-scraper.mjs hashtag funny --no-headless
```

## How It Works

### Profile Scraping
1. **Browser Launch**: Connects to existing Chrome via CDP or launches fresh headless Chrome
2. **Page Navigation**: Visits `https://www.tiktok.com/@{username}`
3. **SSR Extraction**: Parses `__DEFAULT_SCOPE__` JSON from `<script>` tags (TikTok embeds user data server-side)
4. **DOM Fallback**: If SSR fails, reads `[data-e2e="followers-count"]` etc. from DOM
5. **No API calls needed**: All data comes from the page HTML

### Hashtag Video Scraping (CDP Intercept — NOT fetch/XHR)
⚠️ **Key technique**: TikTok's APIs use X-Bogus/X-Gnarly request signatures computed by TikTok's own JavaScript. You CANNOT call these APIs directly via `fetch()` — they return empty bodies. The solution is CDP response interception:

1. **Navigate** to `https://www.tiktok.com/tag/{hashtag}`
2. **TikTok's JS auto-calls** `challenge/detail` + `challenge/item_list` with properly signed requests
3. **CDP intercepts** the responses via `page.on('response', async resp => { await resp.text(); })`
4. **Scroll** the page to trigger TikTok's JS to load more pages
5. Each scroll triggers one more `challenge/item_list` call; CDP captures each batch

### Search Scraping (CDP Intercept)
1. **Navigate** to `https://www.tiktok.com/search?q={keyword}`
2. **TikTok's JS auto-calls** `search/general/full` with signed request
3. **CDP intercepts** the response
4. Scroll to trigger more pages

### User Videos (Auth Required, CDP Intercept)
1. **Load sessionid** cookies from session.json
2. **Navigate** to `https://www.tiktok.com/@{username}` with session injected
3. **TikTok's JS auto-calls** `post/item_list` with signed requests (using sessionid)
4. **CDP intercepts** the responses
5. Scroll to trigger additional pages

## Error Handling

| Exit Code | Meaning | Action |
|-----------|---------|--------|
| 0 | Success | — |
| 1 | General error | Check stderr for details |
| 2 | Login required | Run `auth` command first; open Chrome and log in to TikTok |
| 3 | Profile/hashtag not found | Check spelling; hashtag may not exist |
| 4 | WAF/rate-limit block | Wait 5-10 min; use `--cdp-url` to attach to real Chrome |
| 5 | Session expired | Re-run `node video-scraper.mjs auth` |

## Rate Limiting

TikTok's limits (approximate, without residential proxy):
- **challenge/item_list**: ~100-200 requests per session before temporary rate limit
- **search/general/full**: ~50-100 requests per session
- **recommend/item_list**: ~50 requests per session
- Built-in delays: 1.5-2 seconds between pagination requests

Rate limit symptoms (exit code 4):
- API returns empty `itemList` or `data: []`
- `statusCode` is 10201 or non-zero

If you hit rate limits:
1. **Wait 5-10 minutes** and retry
2. Use `--cdp-url` to attach to a real Chrome session with browsing history
3. Reduce request frequency with `--delay=3000`

## WAF/Bot Detection

TikTok uses PerimeterX. The scraper avoids detection by:
- **Never making direct API calls** (no fetch/curl) — all API calls are made by TikTok's own JS
- **Using CDP response interception** (`page.on('response')`) to capture what TikTok's JS fetches
- Using the browser's real cookies, fingerprint, and X-Bogus/X-Gnarly request signatures
- Using realistic delays between page navigations and scrolls

If you get blocked (exit code 4):
1. Use `--cdp-url=http://localhost:9333` to attach to your real Chrome session
2. Try `--no-headless` to see what TikTok is showing
3. Wait 10-15 minutes and retry
4. Clear TikTok cookies in Chrome and start fresh

**Important**: The `user/detail` and `post/item_list` APIs require `X-Bogus` request signing. The scraper does NOT attempt to call these APIs directly because the signature cannot be forged. Instead:
- Profile data comes from SSR HTML
- User videos are intercepted via CDP after page navigation

## Session Expiry

TikTok sessions last until you log out or TikTok invalidates them (days to weeks typically).

Signs of session expiry:
- `statusCode: 10201` in API response
- Redirect to login page
- Exit code 5

Re-capture session:
```bash
node video-scraper.mjs auth --cdp-url=http://localhost:9333
```

---

## Forced Logout / Security Challenge

TikTok may force re-authentication if it detects suspicious activity:
1. The `user-videos` command detects empty API responses
2. Exit code 2 is returned with a message about needing auth
3. Solution: open Chrome manually, log back in to TikTok, re-run `auth`

## Pagination Details

### Hashtag pagination
- TikTok's JS uses `cursor` field to auto-load next pages as user scrolls
- CDP intercepts each `challenge/item_list` response
- ~30 videos per page (TikTok's default)
- Use `--pages=N` to control how many pages (scroll triggers) to wait for

### Search pagination
- TikTok's JS uses `cursor` field to auto-load next pages as user scrolls
- CDP intercepts each `search/general/full` response
- ~10-12 videos per page
- Use `--pages=N` to control pages

### User videos (CDP intercept)
- Pagination via page scrolling
- TikTok auto-loads next page at bottom of scroll
- CDP captures each `post/item_list` response automatically

## Known Limitations & Caveats

1. **User videos (post/item_list) require auth**: X-Bogus signature cannot be forged. The CDP intercept approach depends on page scrolling triggering real requests.

2. **Video URLs expire**: CDN URLs (`v16-webapp-prime.tiktok.com`) are signed and time-limited (~6-12 hours). Download them promptly.

3. **Avatar/cover URLs expire**: Profile picture and video cover URLs from TikTok CDNs also expire (typically 24 hours).

4. **No comment scraping**: Comment APIs require authentication.

5. **No video download**: The `play_url` in video data is a direct download URL that works while not expired. No additional decryption needed.

6. **Private accounts**: Profile bio/stats are visible; videos and following lists are not without auth + follow relationship.

7. **Geo-restrictions**: Some content may not be visible from certain countries. The browser's IP determines what TikTok shows.

8. **"Something went wrong" on video tab**: This appears on profile pages when TikTok detects the request lacks X-Bogus signature. This does NOT affect profile data extraction (SSR) or hashtag/search scraping.

9. **CAPTCHA**: If TikTok shows a CAPTCHA, the script exits with code 4. Solve it manually in Chrome and retry.

10. **msToken rotation**: The msToken cookie is rotated by TikTok's JS on every page. It's included automatically by the browser when calling APIs from `page.evaluate`. Do not try to use msToken from one session in another.
